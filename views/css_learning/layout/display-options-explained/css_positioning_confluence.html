display:inline

About:

The default value for elements. Think of elements like ‹code›,‹span›, ‹emn›, or ‹b› or ‹a› . Those tags are always displyad in line and it wont break a line unleas you will use ‹br› width and hegiht is IGNORED! if applied to block elements as ‹fiv›or ‹p› the will act as ‹span› margin-bottom and margin-top will be also IGNORED !. inline element wont break a line and the text will jump to the next line if container is too small.

span.inline { display: inline /* it is already there by default*/; width:40px; /*ignored*/ height:40px; /*ignored*/ border: 2px dashed darkgrey; margin-top:40px; /*ignored*/ margin-bottom:40px; /*ignored*/ margin-left:4px; margin-right:50px; /*works*/ }

display:inline-block

About:

inline-block element will be set in row (next to each other) and width and hight , margin-top and bottom will be contrary to display:inline. if the content is bigger than it inline-block contaier it will jump to the next row wit entire block (container): it acts as display:block but inline. By default it will display element on top. it will respect vertical-align:middle,bottom, top if applied.
Example below

span.inline-block ,em.inline-block{ display: inline-block; width: 120px; height: 220px; padding: 3px; border: 2px solid darkgrey; margin:2px; } div.inline-block { display: inline-block; padding: 3px; border: 2px solid pink; width: 40%; margin:2px; } /*no need to for width and height it will adjust to content*/ /* by default it will make a small gap between elements */ /* by default content will be set to top (expected)*/ /* entire element will be set to the bottom*/ }
