        I learn a lot from your posts, thank you!

        I find a problem in the section "Prototypes and Constructors" where you wrote "Interestingly, in both cases the this value will refer to the Constructor object, which will be myConstructor."

        I modified the test code as below, and it printed 'this' value equals variable a.

        var myConstructor = function () {
            this.someMethod = function () {
                console.log('someMethod this:', this);
                console.log('someMethod this === a:', this === a); 
            };
        };

        myConstructor.prototype = {
            somePrototypeMethod: function () {
                console.log('somePrototypeMethod this: ', this);
                console.log('somePrototypeMethod this === a:', this === a); 
            }
        };

        var a = new myConstructor();
        a.someMethod();
        a.somePrototypeMethod();



        In chrome console it will print:

        someMethod this: myConstructor {someMethod: function}
        someMethod this === a: true
        somePrototypeMethod this: myConstructor {someMethod: function}
        somePrototypeMethod this === a: true

        In firefox console it will print:

        someMethod this: Object { someMethod: myConstructor/this.someMethod() }
        someMethod this === a: true
        somePrototypeMethod this: Object { someMethod: myConstructor/this.someMethod() }
        somePrototypeMethod this === a: true


        So, i think the conclusion should be "In both cases this value will refer to the constructed object, which will be a."
